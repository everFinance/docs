# tx

## 功能

所有 everPay 的交易，都会生成 everPay 交易`数据结构`，并通过 `POST` 请求发送至 `tx` API，经过 everPay 后端服务验证。

在用户端，只会生成：
* 转账的 `transfer` everPay 交易。
* 提现的 `burn` everPay 交易。
## 参数
* path `{{endpoint}}/tx`
* Post data `everpayTx`

关于 `Schema` 生成、签名，以及交易记录获取，请阅读 [DOCS - 系统概览 - 交易](../../dive/transaction)。
